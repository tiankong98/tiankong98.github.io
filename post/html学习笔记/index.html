1. HTML基础

1.1新建文档

空白文档

使用<!doctype html>定义文档类型

即网页文档的基本结构，

头部区需添加相关信息，

网页内容添加

文本内容，外部引用内容，各种标签标识内容。

如： 中添加文字

​ 标签自我介绍

1.2头部信息

网页标题

新建文档（head,body）

Head中使用title,

Title作用：

定义标题栏的标题，

定义收藏夹的列表标题，

定义搜索结果的列表标题（搜索引擎依据）

Title和H1比较：

Title： 网页标题，搜索检索，唯一且必要，head区域

H 1: 文档标题，用户浏览，不唯一不必要，body区域

网页元信息
Meta标签 （meta.html）

元信息：不是提供给用户的信息，用户不可见

提供浏览器解析，方便搜索引擎对页面信息的检索。

meta类型：

http-equiv(equivalence):向浏览器传有用的信息，帮助浏览器正确地显示内容。类似http头部消息，先被浏览器获取

Name:描述网页元信息，存在与网页中，晚于http-equiv解析。

文档视口 (viewport.html)

分辨率不同，显示不同

Viewport: 可视区域

移动前端： 设备浏览器 显示页面内容的屏幕区域

Dip(device-independent pixel,设备逻辑像素)

CSS像素

Visual viewport （视觉视口） 物理像素

Ideal viewport（理想视口） 逻辑像素（分辨率）

Layout viewport（布局视口） CSS像素

视口标签语法：

height=device-heigth,

initial-scale=1.0,初始缩放比率，逻辑像素和CSS像素一致

minimum-scale=1,

maximum-scale=1,

user-scalable=no”>

1.3 基本结构

定义文档结构

Html head(meta title) body(h p ul li)

div

​ Div:定义文档结构块

​ Span：包裹行内内容或元素，部分文字标识

div是一个通用容器，没有任何语义，用户可以为其添加样式或者js效果。在h5之前，div包着大块内容（如页眉、页脚、主要内容等）。div可以作为h5的备用容器使用。

id class

Id唯一，class标识类样式，可不唯一

id可以用来标识持久的结构性元素，列入主导航和内容区域

与id不同，同一个class可以应用与页面上任意数量的元素，class非常适用于标识样式相同的对象。

或：

上述例2删除不必要的类有助于简化代码，使页面更简洁。

title

为相关元素定义额外信息，鼠标经过会有提示。（超链接、图像中经常使用）

href=Hypertext Reference

role

告知搜索引擎或其他设备当前元素的角色。

Role=banner/navigation/check-box/main

即使不适用role,页面看起来也没有任何差别，但是使用它们可以提升使用辅助设备的用户体验。

html注释

浏览器不会解析注释的信息，方便阅读代码

1.4 主结构

页眉

header表示页眉，用来标识标题栏，功能是引导和导航作用的结构元素。

导航

nav表示导航条，用来标识页面导航的链接组，一个页面可以有多个nav: 主菜单导航、侧边栏导航、页内导航，翻页操作等。

main

main表示主要区域，标识网页的主要内容，对于文档来说应当是唯一的.

article

article表示文章块，用来标识页面中完整独立的内容。

section

section表示区块，用于标识文档中的节，主要用于对内容进行分区。

从语义上讲，section标记的是页面中的特定区域，而div则不传达任何语义。

aside

aside表示附栏，用来标识所处内容之外的内容。可以包含当前页面或主要内容相关的引用，侧边广告等。

footer

footer定义页脚，表示标注，通常包含它所在区块的信息，如指向相关文档的链接、版权信息、作者等。

2.文本

2.1标题和正文

标题

一级到六级标题

语义，标题信息相对内容跟重要，

层级重要性逐渐递减，使用中注意与普通文本的区分，标题文本嵌套结构要注意层级关系。

一般用到4级标题（子栏目或者版权信息等），5、6比较少用

段落

包裹段落文本，具有默认样式。

行距一般为1.1em,每段间隔12px

备选

6.3设置属性

名称和值

表单控件有name(服务器根据name获取数据) value

布尔型属性

Input属性控制表单控件的状态

Readonly 只读，可获取焦点，不可写，主要用于文本框。 Disabled:禁用，用于文本框和按钮，数据不会提交。

Require:必填，多用于文本框，checked:选中，多用于单选按钮，复选框；selected:选择，下拉菜单select或者数据列表datalist，autofocus:自动获取焦点，multiple:多选，用于列表框，文件域等。

必填

Require，输入框不能为空

禁止验证

Novalidate=true,formnovalidate=true

多选

Input type=file时只支持一个文件上传，

Multiple=”multiple”可同时上传多个文件（部分浏览器支持）

自动完成

Html5 Autocomplete(与文本有关的表单类型都可设置)

Autocomplete=”on” on,off
